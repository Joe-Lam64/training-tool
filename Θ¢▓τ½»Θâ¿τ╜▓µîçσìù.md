# 🚀 雲端部署 Step-by-Step 指南

> 目標：讓你拿到一個網址（例如 https://your-tool.onrender.com），同事打開就能用，**不用你電腦開著**

---

## 📋 整體流程預覽

```
你的電腦 (zip 解壓)
    ↓ ①
    把程式碼上傳到 GitHub
    ↓ ②
    GitHub 變成你的「程式碼倉庫」
    ↓ ③
    在 Render 連結這個倉庫
    ↓ ④
    Render 自動部署 → 給你一個網址
    ↓ ⑤
    把網址給同事 (任何電腦都能用)
```

預估時間：**20-30 分鐘**（第一次比較慢，之後改動只要 5 分鐘）

---

## ✅ 開始前檢查清單

- [ ] 已下載並安裝 **Git for Windows** ([下載](https://git-scm.com/download/win))
  - 安裝時全部用預設值「Next」就好
  - 安裝完開 cmd 輸入 `git --version` 應該看到版本號
- [ ] 已註冊 **GitHub 帳號** ([註冊](https://github.com/join))
- [ ] 已下載這版 zip 並解壓到桌面

---

## 第一步：在 GitHub 建立「程式碼倉庫」

1. 打開瀏覽器到 https://github.com
2. 登入後，點右上角的 **+** → 選 **New repository**
3. 填寫：
   - **Repository name**: `training-tool` （或你喜歡的名字）
   - **Description**: 外訓課程整合工具（選填）
   - **Public** ⚠️ 選這個（Render 免費版要求公開倉庫）
   - 其他不要勾選
4. 點 **Create repository**

✅ 你會看到一個畫面，**先不要關**，待會要用到

---

## 第二步：把程式碼上傳到 GitHub

### 2-1. 開啟 Git Bash

1. 用檔案總管打開你解壓出來的 `training_tool` 資料夾
2. 在資料夾**空白處**按右鍵 → 選 **Open Git Bash here**
   - 如果沒看到這個選項，就在資料夾上方網址列輸入 `cmd` 按 Enter

### 2-2. 輸入這幾個指令（複製貼上一次一行，按 Enter）

```bash
git init
git add .
git commit -m "Initial commit"
```

如果跳出「Please tell me who you are」錯誤，就先輸入這兩行（換成你的資訊）：
```bash
git config --global user.email "你的Email@gmail.com"
git config --global user.name "你的GitHub名稱"
```
然後重新做 2-2。

### 2-3. 連到你的 GitHub 倉庫

回到 GitHub 那個畫面（第一步），複製中間那段「**…or push an existing repository from the command line**」的指令，**整段複製貼到 Git Bash**：

```bash
git remote add origin https://github.com/你的帳號/training-tool.git
git branch -M main
git push -u origin main
```

第一次推送會跳出登入視窗，輸入 GitHub 帳號密碼（或用 Personal Access Token）。

### 2-4. 確認上傳成功

重新整理 GitHub 那個頁面，應該會看到你的 `app.py`、`Procfile`、`requirements.txt` 等檔案 ✅

---

## 第三步：到 Render 註冊並部署

### 3-1. 註冊 Render

1. 打開 https://render.com
2. 點 **Get Started** → 選 **Sign Up with GitHub**
3. 授權 Render 存取你的 GitHub

### 3-2. 建立 Web Service

1. 登入 Render 後，點左上角的 **+ New** → 選 **Web Service**
2. 在「Connect a repository」找到你剛建立的 `training-tool`，點 **Connect**

### 3-3. 設定部署選項（重要！）

| 欄位 | 填什麼 |
|------|--------|
| **Name** | `training-tool`（會變成網址的一部分） |
| **Region** | Singapore (新加坡，離台灣最近) |
| **Branch** | main |
| **Runtime** | Python 3 |
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120` |
| **Instance Type** | **Free** （免費方案） |

3. 滾到最下面，點 **Create Web Service**

### 3-4. 等候部署完成

- 你會看到一個畫面跑著 log（黑色文字滾動）
- 第一次部署需要 **3-5 分鐘**
- 看到「Your service is live 🎉」就成功了

✅ 上面會顯示你的網址：`https://training-tool-xxxx.onrender.com`

---

## 第四步：測試網址

1. 點剛剛的網址（或複製到瀏覽器）
2. 應該會看到登入頁面 📚
3. 用 `train0` / `HR123` 登入
4. 試試「立即更新」抓取課程

---

## 第五步：給同事用

直接把網址貼給同事：
```
網址：https://training-tool-xxxx.onrender.com
帳號：train1 / 1234
```

**他們完全不需要**：
- ❌ 不用裝 Python
- ❌ 不用下載任何檔案
- ❌ 不用連你的電腦
- ❌ 不用在公司網路

只要有網路、有瀏覽器就能用 ✨

---

## 🔄 未來如何更新功能？

當我給你新版的 `app.py`：

1. 把新檔案覆蓋舊的
2. 開 Git Bash，輸入：
   ```bash
   git add .
   git commit -m "update features"
   git push
   ```
3. Render 會**自動偵測**並重新部署（3-5 分鐘）
4. 同事重新整理瀏覽器就看到新版

---

## ⚠️ Render 免費版的限制

1. **休眠機制**：15 分鐘沒人用，服務會「睡覺」，下次訪問要等 30-60 秒才會醒
   - 解法：升級付費版（$7/月）就不會睡
   - 或：你跟同事約定使用時段，每天定時開幾分鐘預熱
2. **資料不持久**：免費版重啟時，`users.db` 和 `course_data.json` 會被清掉
   - **解法 1**：把預設帳號寫死在 `app.py`（每次重啟自動建立）✅ 已經這樣做了
   - **解法 2**：每次用之前先按「立即更新」抓最新課程
3. **每月用量上限**：750 小時/月（差不多就是不間斷使用）

## 🆘 遇到問題怎麼辦？

**狀況 A：git push 跳「Authentication failed」**
- 用 Personal Access Token：到 GitHub → Settings → Developer settings → Personal access tokens → 產生一個，貼到密碼欄

**狀況 B：Render 部署失敗（Build failed）**
- 看 Logs 顯示什麼錯誤，截圖給我看

**狀況 C：網址打開是 502 / 503 錯誤**
- 服務在睡覺，等 30-60 秒重試

**狀況 D：登入後按「立即更新」沒反應**
- Render 免費版 timeout 較短，等 15-20 秒看看
